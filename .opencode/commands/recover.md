# 灾难恢复

你的任务是从文件损失中恢复。严格遵循 `AGENTS.md` 的"灾难恢复"章节。

执行以下步骤：

1. 如果用户提供了损失报告路径，读取它；否则让用户提供
2. 提取损失文件清单
3. 逐文件交叉对比**当前对话**中是否出现过（被编辑/创建/讨论过）
4. 按三级策略恢复，输出恢复报告

## 关键约束

- **绝不凭空生成**：对话中从未出现的文件 → 直接标记不可恢复
- **git 优先**：跟踪文件先从 `git show HEAD:<路径>` 获取基准版本
- **上下文验证**：恢复后逐文件对比内容是否与对话描述一致
